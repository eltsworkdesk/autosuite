Artifact — Accessibility Checklist

WCAG AA baseline

Applies to both the consumer site and the dealer OS.

Keyboard & focus
☐ Every interactive element reachable via Tab in logical order
☐ Visible focus ring on all buttons, inputs, cards, table rows (not just outline:none)
☐ Kanban drag-and-drop has a keyboard equivalent (move card via menu/shortcut)
☐ Command palette (⌘K) fully keyboard-operable, Esc closes
☐ Modals trap focus and return focus to trigger on close
Screen readers
☐ Status pills (New/Sold/etc.) have text, not color alone, read by SR
☐ Charts (funnel, revenue bars) have a text/table alternative
☐ Form fields have associated <label>, errors announced via aria-live
☐ Icon-only buttons (⚙ 🔔 ♡) have aria-label
☐ Toasts use aria-live="polite" so updates are announced
Contrast
☐ Body text ≥ 4.5:1 against background (verify oklch grays at final render)
☐ Status pill text meets 4.5:1 against its tinted background, not just against white
☐ Placeholder text is not the only indication of a field's purpose
☐ Dark sidebar inactive nav text (currently ~60% opacity) meets 4.5:1 minimum
Touch & responsive
☐ All tap targets ≥ 44×44px on mobile (bottom tab bar, card actions)
☐ Text reflows without horizontal scroll down to 320px width
☐ Pinch-zoom never disabled
Forms
☐ Financing/trade-in/test-drive forms show inline errors next to the field, not just at top
☐ Required fields marked programmatically (aria-required), not color alone
☐ Sliders (financing calculator) have accessible name + current value announced
